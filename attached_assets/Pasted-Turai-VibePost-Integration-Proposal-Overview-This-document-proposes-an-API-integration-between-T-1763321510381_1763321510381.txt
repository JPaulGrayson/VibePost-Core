Overview

This document proposes an API integration between Turai (AI-powered location tours) and VibePost (social media automation platform) to enable automated, trend-responsive content generation.

The Concept: VibePost identifies trending topics and relevant conversations across X, Reddit, and Discord. Turai generates location-based visual postcards that connect those trends to real places, which VibePost can then share across platforms.

Use Cases

1. Trend-Responsive Content

Trend: "Solar eclipse 2024" trending on X
VibePost: Identifies the trend and relevant locations (Carbondale, IL)
Turai: Generates postcard for Carbondale with eclipse-themed narrative
VibePost: Posts to X/Instagram with trending hashtags
2. Event-Based Marketing

Trend: "Taylor Swift Eras Tour Nashville"
Turai: Creates Nashville tour postcard with Swiftie angle
VibePost: Replies to relevant conversations with visual content
3. Seasonal Content

Trend: "Best fall foliage 2024"
Turai: Generates postcards for Vermont, Smoky Mountains, etc.
VibePost: Shares across multiple platforms with platform-optimized aspect ratios
4. Location Discovery

Trend: "Hidden gems in Utah"
Turai: Creates postcards for lesser-known Utah locations
VibePost: Posts as content series across social platforms
Proposed API Design

Endpoint

POST /api/postcards/generate-by-topic

Request Format

{
  "location": {
    "name": "Nashville, TN",
    "coordinates": {
      "lat": 36.1627,
      "lng": -86.7816
    }
  },
  "topic": "Taylor Swift Eras Tour",
  "aspectRatio": "1:1",
  "stylePreset": "vibrant",
  "contextHint": "Focus on music venues and cultural significance"
}

Request Parameters

Parameter	Type	Required	Description	Example Values
location.name	string	Yes*	Human-readable location	"Nashville, TN", "Route 66"
location.coordinates	object	Yes*	Lat/lng coordinates	{lat: 36.16, lng: -86.78}
topic	string	Yes	Trending topic or theme	"Solar eclipse", "Fall foliage"
aspectRatio	string	No	Social platform format	"1:1", "9:16", "4:5", "16:9", "1.91:1"
stylePreset	string	No	Visual style	"vibrant", "classic", "minimal"
contextHint	string	No	Additional context for AI	"Focus on family activities"
*Either location.name OR location.coordinates required (or both)

Response Format

{
  "success": true,
  "data": {
    "imageUrl": "https://turai.com/postcards/abc123.png",
    "caption": "Experience Nashville like a true Swiftie! ðŸŽ¸ Explore the music venues that shaped her career...",

Error Response

{
  "success": false,
  "error": {
    "code": "GENERATION_FAILED",
    "message": "Unable to generate postcard for the specified location",
    "details": "No relevant location data found"
  }
}

Authentication Options

Option 1: API Key (Recommended)

VibePost includes API key in request header: X-API-Key: your_key_here
Simple, secure, trackable
Allows rate limiting per integration partner
Option 2: Public Endpoint with Rate Limiting

No authentication required
Global rate limit (e.g., 100 requests/hour per IP)
Simpler integration but less control
Option 3: OAuth (If needed)

Full OAuth flow for more complex integrations
Probably overkill for this use case
Question for VibePost: Which authentication approach works best for your architecture?
